# Meta Ads网页广告的投放配置说明

Source URL: https://support.appsflyer.com/hc/zh-cn/articles/25646834966033
Section: Meta ads
Updated: 2026-06-23T12:48:58Z

概要： 您可以使用Meta Web对接对Meta Ads带来的网页到应用转化进行归因。

## 速览

AppsFlyer支持网页端获客广告专用的Meta Ads对接（ metaweb_int ），您可以通过Meta的Conversion API（CAPI）上报相关广告的归因和转化数据。然后，Meta就可以利用这些数据来优化定向投放、衡量广告效果并打造自定义人群包。

#### 注意

目前，Meta Web对接尚不支持通过CAPI发送再营销回传数据。

### 前期准备

在启用Meta Ads对接前，请确保您已在 Meta Business Manager 中开设了企业账户。

### 如何创建Meta Web广告活动

本文说明了Meta Ads中网页广告的设置方式，适用于通过AppsFlyer衡量投放效果的移动端、PC端和主机端应用。设置流程包括：将Meta Ads与AppsFlyer对接、在两个平台上完成广告活动配置，并使用OneLink Smart Script或OneLink Smart Banners创建落地页。
该流程中必须完成的步骤如下：

- 在AppsFlyer中设置Meta Web对接 （必须完成）

- 在Meta Ads中设置AppsFlyer对接 （必须完成）

该流程中的可选步骤如下：

- 创建带有OneLink智能脚本的落地页 （可选）

- 创建带有OneLink智能横幅的落地页 （可选）

## 在AppsFlyer中设置Meta Web对接

请先在AppsFlyer后台启用Meta Web对接，并设置AppsFlyer向CAPI发送回传的方式。

### 1. 激活Meta Web广告对接

请按以下方式启用Meta Web对接：

- 在AppsFlyer面板的左侧菜单栏中点击 协作 > Partner Marketplace 。

- 搜索并选择 Meta Web 。

- 点击 设置对接 ，然后界面会跳转到 活跃对接 设置页面。

- 在 渠道对接 选项卡中，打开 启用该渠道 开关。 请注意： 在您与该渠道合作期间，请确保该开关总是处于打开（即 启用 ）状态。

- 关闭 高级隐私保护开关 （设置iOS应用对接时适用）。

- 填写 Pixel ID 字段。您可以从Meta ads Manager后台获取Pixel ID信息，具体方式如下：
- 在 Ads manager 侧边栏菜单中，向下滚动到 All tools > Events Manager 。 请注意： 请确保您在右上角的下拉菜单中选择的业务（Business）正确无误。

- 从 Events Manager 侧边栏菜单中选择 Data sources 。您会看到一个pixel列表。 如果您选定的业务尚无pixel，可以在该窗口中创建一个pixel。

- 复制该pixel名称下方的pixel ID，
并将其粘贴到
AppsFlyer面板中的 Pixel ID 字段。

- 填写 Access Token 字段。您可以按照 这里的说明 来获取该token信息。

- 打开 限制Meta Ads对用户个人信息的使用（CCPA） 开关，以确保数据的使用方式符合相关的 数据保护法 规定。 请注意： CCPA隐私条例限制他人使用加州的用户所产生的数据，因此CCPA合规后，Meta Ads上这类用户的数据可用性有限。

### 2. 启用浏览型归因

AppsFlyer支持通过启用 智能脚本 或 智能横幅 的落地页，对Meta Web广告系列进行浏览型归因。这项功能可以基于概率模型(PMOD)，将因广告曝光而触发的App激活进行归因，即使用户并未直接点击广告。

当用户点击Meta Web广告并进入 智能脚本 或 智能横幅 落地页时，AppsFlyer可以将该活动与后续的应用安装进行匹配。如果可用，回传中将包含 fbclid 以提高归因准确性。

如需启用Meta Web的浏览型归因:

- 在AppsFlyer中，从侧边栏打开 活跃对接 > Meta web ads 。

- 在 对接 选项卡中，下滑到 激活归因 部分。

- 打开 浏览型激活归因 开关。

- [可选] 开启 跨平台归因具有同等优先级 并设置回溯窗口。有关更多信息，请参见 为跨平台展示启用同等优先级 。

- 打开 归因链接 选项卡。

- 下滑到 浏览型归因 部分并设置 浏览归因回溯窗口 （最长24小时）。

- 点击 保存对接 以应用您的设置。

### 2. 应用内事件回传

目前只有用户激活才能触发自动的Meta Ads默认回传。如需回传其他类型的事件，可以将AppsFlyer事件映射到Meta Ads事件（请在 Meta Events Manager 中查看Meta侧的事件名称信息），从而创建自定义回传。
请按以下方式设置应用内事件回传：

- 打开 应用内事件回传 开关。

- 根据实际需求设置应用内事件回传窗口。详情请见 应用内事件回传窗口配置 。

- 点击 添加事件 ，将相关事件添加到列表中。

- 在 AppsFlyer事件 一列，从下拉菜单中选择相关的事件名称。AppsFlyer会通过已接入您应用的SDK或S2S事件来获取该事件的名称信息。

- 在 渠道映射事件 一列 ， 从下拉菜单中选最合适的渠道侧预定义事件，或选择自定义事件（CUSTOM）。

- 在 事件来源 一列，选择 仅限该合作渠道 ，确保仅回传Meta Ads带来的用户所完成的事件。

-
在 包括 列中，选择是否将事件与其收入值一起发送。请见 回传中包含的数据 。

#### 注意

- 如果您设置的是购买事件，请将 包含 列设为 值和收入 。选择其他选项将导致回传失败。

- 目前，仅购买事件支持包含收入。

- 【可选】点击 添加条件 图标，设置事件的 映射条件 。

- 点击 保存对接 。

- 【可选】您可以在保存对接后留在配置页面，将其他应用也对接到该渠道：
- 点击页面左上角渠道名称下方的应用名称，打开应用列表。

- 在下拉菜单中选择一个其他应用。

- 为选定应用完成对接流程。

## 在Meta Ads中配置AppsFlyer对接

本节所述步骤主要在Meta Ads Manager中完成。

### 1. 设置投放类型、投放目标和转化目标

- 登入Meta Ads并创建一个广告系列。

- 在 Campaign objective 部分中选择以下任一选项：
- Sales ：该选项能让Meta针对某个具体事件进行优化（比如您的主要KPI事件）。

- Traffic ：该选项能让Meta针对广告点击进行优化。建议您仅在没有其他KPI或其他KPI事件的流量太低时使用该选项。

- 【可选】如果您想要在上述选项中添加自定义的投放目标，请按以下方式操作：
- 在Meta Ads中选择一个要作为投放目标的自定义事件，并将其标记为转化行为（conversion action）。

-
进入AppsFlyer面板，在 Meta Web 对接页面中的 应用内事件 部分选择一个AppsFlyer事件，将其映射到渠道侧的自定义事件（在 渠道映射事件 一列中选择CUSTOM），并使用Meta Ads中的转化行为名称（conversion action）为其命名。

#### 注意

如需进一步了解Meta的投放目标（Campaign Objectives），请参阅 Meta提供的相关说明 。

- 将 转化位置（conversion location） 设置为 Website 。

- 请确保 Pixel 字段的值与您在AppsFlyer中为Meta Web对接（ metaweb_int ）所配置的Pixel ID完全一致。如需进一步了解Meta Pixel，请参阅 Meta提供的相关说明 （需登录）。

- 点击 Next 。

### 2. 设置目标网址（Destination URL）

-
在 Destination 下的 Website URL 部分中 ， 输入相关URL，作为用户点击广告后访问的页面。请将此处的URL设置为以下任一网址：

-
落地页 - 适用于以下场景：

- 您的应用在多个平台发布。

- 您希望通过落地页更详细地介绍您要推广的产品。

- 您希望通过落地页收集更多数据。

在以上所有场景中都可以使用带有 智能脚本 或 智能横幅 的落地页。

-
AF归因链接 ：如果您不想让用户访问落地页，而是直接跳转到商店并下载相关应用，建议您使用AF归因链接（包括OneLink链接、单平台链接和跨平台链接）作为目标网址。

#### 重要提示！

部分情况下，AF归因链接无法正常生效。如果发生这种情况，请向您的Meta客户经理咨询解决方案，或直接使用落地页作为目标网址。

- 点击 Build a URL parameter ，将链接参数添加到该URL中（落地页地址或AF归因链接都适用）。

-
在 Custom parameters 下点击 Add parameter 。

-
将AppsFlyer参数映射到Meta中对应的值。

-
如果您的目标网址是 落地页、OneLink归因链接或单平台归因链接 ，请将以下AF参数映射到Meta侧的对应参数：

### 3. 选择需要监测的事件（tracked events）

## 创建带有OneLink智能脚本的落地页

用户链路： 用户点击Meta广告后跳转到您的移动端网站，并进入一个带有CTA按钮的网页，引导用户从一个或多个平台下载您的应用。用户点击该按钮后，就会跳转到相应的商店。

### 整个链路中的数据流

- Meta广告的URL中包含 pid=metaweb_int 参数。当用户点击该广告时， fbclid 参数会被自动添加到该广告URL的末尾。

- 用户跳转到指定的目标网址——即一个带有智能脚本的落地页。

- 在该落地页加载时，智能脚本会向AppsFlyer发送一个展示，其中包含从Meta传入落地页的URL参数。AppsFlyer可通过该展示对落地页进行浏览型归因。如需进一步了解广告展示和浏览型归因，请参阅 此文档 。

- 智能脚本会根据传入链接中的参数，为落地页的CTA按钮生成传出链接，其中包含 fbclid 参数（这里的传入链接是指从广告进入落地页的链接，传出链接是指从落地页跳转到其他地址的链接）。这些按钮可以引导用户：
- 进入Google Play和App Store（通过OneLink归因链接实现）。

- 进入多个平台的商店（例如CTV、Epic和Steam）。在此场景中，智能脚本会针对各个平台生成一条单平台归因链接，用户点击后会跳转到对应平台的商店。

- 在用户设备中下载APK文件。

- 用户点击CTA按钮后，AppsFlyer会收到并记录该次点击。

- 用户跳转到相应的商店并下载该应用。

- 用户打开该应用时，AppsFlyer会记录到一个激活。

- 如果激活数据中带有可用的 install referrer ，AppsFlyer会根据概率模型或确定性归因逻辑对这个激活进行归因。

- AF会生成一条带有 fbclid 参数以及其他事件参数的回传，并通过CAPI将其发送到Meta。Meta通过 fbclid 参数来识别相关点击，并使用其他参数对具体广告系列的投放进行优化。

### 设置带有智能脚本的落地页

如需在您的Meta Ads广告投放中使用智能脚本，请按以下方式操作：

- 为您要投放的广告系列设置一个OneLink模板，用于生成相关的OneLink归因链接。详情请见 OneLink模板的创建方式 说明。

-
创建一个智能脚本。详情请见 OneLink智能脚本的创建方式 说明。

#### 注意

下文内容主要针对使用智能脚本的Meta Ads广告系列，说明了具体的设置步骤。如需了解更具普适性的智能脚本投放设置方式，请参考 OneLink智能脚本V2设置说明 中的 OneLink智能脚本创建方式 部分。

- 请确保所有相关的URL参数都已添加到传入链接末尾。如需检查附加在传入链接末尾的参数，请打开Meta Ads Manager，查看您在创建目标网址（Destination URL）时设置的URL参数。详情请见 设置目标网址（Destination URL） 。

- 将Meta Ads广告中必须配置的传入URL参数映射到由脚本生成的传出URL参数，用户点击CTA按钮时，智能脚本会将这些参数发送到AppsFlyer：
- PID ：确保智能脚本从传入链接中提取 pid=metaweb_int 参数，并将其映射到传出链接中对应的参数。

-
FBCLID :为确保应用内事件的回传能成功发送到Google Ads，传出链接中必须带有 fbclid 参数。传出链接中的FBCLID参数映射取决于您使用的智能脚本版本。

- 2.8.1及以上版本：传入链接中带有 fbclid 参数时，其参数值会自动传递到传出链接的对应参数中，无需手动映射。

- 2.8.0及以下版本：必须在 智能脚本 页面的 URL参数映射 部分手动配置 fbclid 参数的映射。
- 在 映射对象 一栏中选择 自定义参数 。

- 在 出链参数 和 入链参数 字段中输入 fbclid 。

#### 注意

如需使相关数据呈现在AppsFlyer的原始数据报告中，必须将传入的 fbclid 参数同时映射到传出的 fbclid 参数和 af_sub[1-5] 参数。详情请见 Facebook FBCLID的说明 。

### 设置带有智能脚本的跨平台落地页

如需在您的Meta Ads广告投放中使用带有智能脚本的跨平台落地页，请按以下方式操作：

- 为您要投放的广告系列设置一个OneLink模板，用于生成相关的OneLink归因链接。详情请见 OneLink模板的创建方式 说明。
- 请参考 此文档 中的说明，将您的OneLink模板应用于跨平台归因。

- 请确保所有相关的URL参数都已添加到传入链接末尾。如需查看附加在传入链接末尾的参数，请打开Meta Ads Manager，检查您在创建目标网址（Destination URL）时设置的URL参数。详情请见 目标网址（destination URL）的设置说明 部分。

- 请列出各传入参数在传出链接中的对应参数。例如将传入链接中的 af_site_id 参数映射到传出链接中的 af_site_id 参数。

- 请列出必要传出参数（即不能出现空值的参数）的默认值。

- 向您的开发人员发送以下信息：
- 您的传入-传出参数的映射方式。开发人员会创建一个脚本，根据您指定的传入-传出参数的对应关系，生成传出的直接访问链接。

- 各参数的默认值。脚本若未在传入的链接中发现对应的参数值，就会使用默认值来填充传出链接中的参数。

- 如果您使用OneLink归因链接，请将相关域名和模板ID发送给您的开发人员，用于创建传出的OneLink链接。域名和模板ID的格式为 yourbrand.onelink.me/A1b2 ，若您使用品牌域名，则为 click.yourbrand.com/A1b2

- 将以下Dev Hub文档发送给您的开发人员：
- 创建可直接访问非移动端页面的传出链接时，请参考这些 说明及代码示例 。

- 创建跨平台落地页时，请参考这些 演示和说明 。

-
提醒开发人员务必在传出链接中加入传入参数 fbclid 的映射。

#### 注意

如需使相关数据呈现在AppsFlyer的原始数据报告中，必须将传入的 fbclid 参数同时映射到传出的 fbclid 参数和 af_sub[1-5] 参数。

## 创建带有OneLink智能横幅的落地页

用户链路： 用户点击Meta广告后跳转到您的移动端网站，并进入一个展示智能横幅的页面，该横幅中带有CTA按钮，引导用户从对应的应用商店下载相关应用。

### 整个链路中的数据流

- Meta广告的URL中包含 pid=metaweb_int 参数。当用户点击该广告时， fbclid 参数会被自动添加到该广告URL的末尾。

- 用户跳转到指定的目标网址——即一个带有智能横幅的落地页。

- 落地页加载的同时，智能横幅会向AppsFlyer发送一条 展示 ，其中包含传入链接参数（传入链接是指从广告进入落地页的链接）。有了这条展示数据，AppsFlyer就能通过浏览型归因将转化匹配到落地页。如需进一步了解广告展示和浏览型归因，请参阅 此文档 。

- 智能横幅针对该横幅中的CTA按钮，生成传出的URL。生成的传出链接以传入链接中的参数（包括 fbclid ）为基础。智能横幅只能生成OneLink链接。

- 当用户点击智能横幅中的CTA按钮时，AppsFlyer就会接收并记录该次点击。

- 用户跳转到相应的商店并下载该应用。

- 用户打开该应用时，AppsFlyer会记录到一个激活。

- 如果激活数据中带有可用的install referrer，AppsFlyer会根据概率模型或确定性归因逻辑对这个激活进行归因。

- AF会生成一条带有 fbclid 参数以及其他事件参数的回传，并通过CAPI将其发送到Meta。Meta通过 fbclid 参数来识别相关点击，并使用其他参数对具体广告系列的投放进行优化。

## 常见问题

### 为什么我看到激活和应用内事件归因为Meta Ads（facebook_int），而不是Meta Web（metaweb_int）？

这是由于您在 Meta Ads 中 设置追踪事件 时，将其设置为应用事件而非网页事件。请将设置改为网页事件。

特点 说明

成本数据 系统不会自动获取Meta Ads（ metaweb_int ）网页广告的成本数据。您需通过以下两种方式之一 手动导入：通过电子邮件上传文件，或通过AppsFlyer主机端界面手动上传。 此规则适用于所有用户，包括ROI360订阅用户。
