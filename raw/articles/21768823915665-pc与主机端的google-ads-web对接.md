# PC与主机端的Google Ads Web对接

Source URL: https://support.appsflyer.com/hc/zh-cn/articles/21768823915665
Section: Google Ads
Updated: 2026-06-02T18:43:30Z

概要： 通过Google Ads网页广告活动推广您的PC及主机端应用，实现归因分析。AppsFlyer支持使用跨平台落地页或直接归因链接的Google网页广告。

#### 注意

本文重点关注用于推广PC或主机端应用的网页广告活动。如需创建推广移动应用的网页广告活动，请参阅 Google Ads（AdWords）- 网页端广告活动 。

## 速览

本文详细说明了PC和主机端应用在AppsFlyer中与Google Ads Web对接的流程及其使用方式。其中包含以下信息；

- 在AppsFlyer中启用合作渠道对接 。

- 在AppsFlyer中配置Google Ads Web对接 。

- 在Google Ads中设置PC和主机端应用广告 。

### 前期准备

在启用Google Ads Web对接之前，请确保您已具备以下条件：

- 一个PC或主机端应用

- 具备一个AppsFlyer账户

- 具备必要 OAuth权限 的Google访问令牌。您可在Google Developer Console中验证访问令牌的权限范围。

- 设置该对接的AppsFlyer 用户具有必要的角色/权限 ，并有权访问相关应用

## 在AppsFlyer中激活合作渠道

请按以下步骤启用该渠道：

- 在AppsFlyer面板的左侧菜单栏中点击 协作 > Partner Marketplace 。

- 搜索并选择 Google Ads Web 。

- 点击 设置对接 ，然后界面会跳转到 活跃对接 设置页面。

- 在 渠道对接 选项卡中，打开 启用该渠道 开关。
请注意： 在您与该渠道合作期间，请确保该开关总是处于打开（即 启用 ）状态。

- 参考下文说明，根据您的具体需求在相关选项卡中完成各项配置。

## 在AppsFlyer中配置Google Ads Web对接

这部分还包括Google Ads的回传配置。Google Ads Web（googleads_int）对接通过Google Ads为广告主提供的API工具，将相关的 PC和主机端应用数据 直接发送到Google的服务器。这类数据包括用户在非移动端（包括网页和应用）完成的购买、注册等等各种转化行为。然后Google Ads会使用这些回传数据对广告投放进行优化和定向、衡量广告效果，并创建用于再营销的自定义人群包。

Google Ads Web对接仅支持激活和应用内事件的回传。

### 渠道对接选项卡

渠道对接 选项卡中包括以下几个部分。

#### 启用渠道

在配置对接之前，必须先打开 启用该渠道 开关。

#### 通用设置

#### 注意

将在您使用Google登录并允许AppsFlyer获取 token（见下方步骤），“令牌缺失”的错误提示将不再显示。如果在登录后错误仍然存在， 请验证您的令牌是否具有所需的权限 。

如需设置iOS应用对接：关闭 高级隐私 。

- 首次打开转换ID ：输入您在Google广告中设置的 First open 转化操作ID。如需查看该ID，
- 请进入Google Ads后台并打开 First open 转化操作页面中的 Details 选项卡。

- 在该页面URL中复制 ctid=* 查询参数的值。举例来说，假设 ctid=123456789 ，您需要将 123456789 添加到 First open conversion action ID 字段中。

- 输入您的 Customer ID 。您可以从Google Ads Manager中获取该ID：
- 登入您的Google Ads账户。

- 点击界面右上角的Google账户图标，然后向下滚动到Customer ID部分。

- 如果您有多个Google Ads账户同时为相关应用投放广告，必须使用经理账号（MCC）的Customer ID才能进行跨账户的转化监测。详情请见 《Google提供的MCC账号配置说明》 。

- 如需允许AppsFlyer将转化数据上传到Google Ads，请点击 使用Google登录 。

- 在Google中，选择MCC账号（即您在以上步骤中获取到Customer ID的MCC账号）中的管理员。

- 点击 Allow 。

#### 启用浏览型归因

AppsFlyer支持通过启用 智能脚本 或 智能横幅 的落地页，对Google Web广告系列进行浏览型归因。这项功能可以基于概率模型(PMOD)，将因广告曝光而触发的App激活进行归因，即使用户并未直接点击广告。

当用户看到Google广告并访问启用了智能脚本或智能横幅的落地页时，AppsFlyer可以将后续发生的App激活归因至最初的网页曝光事件，包括跨平台场景。

在可用的情况下，AppsFlyer会在给Google的回传中包含gclid参数。

启用浏览型归因

- 在AppsFlyer中，从侧边栏打开 活动对接 > Google Ads Web 。

- 在 对接 选项卡中，下滑到 激活归因 部分。

- 打开 浏览型激活归因 开关。

- [可选] 开启 跨平台归因具有同等优先级 并设置回溯窗口。有关更多信息，请参见 为跨平台展示启用同等优先级 。

- 打开 归因 选项卡。

- 下滑到 浏览型归因 部分并设置所需的 浏览归因回溯窗口 （最长24小时）。

- 点击 保存对接 以应用更改。

#### 默认回传

激活回传自动发送到Google Ads Web，不会发送到其他渠道。

#### 应用内事件回传

针对发送到Google Ads Web的应用内事件配置回传映射。您可以在Google Ads Manager中查看这些事件。

- 打开 应用内事件回传 开关。

- 点击 添加事件 ，将SDK事件或S2S事件添加到列表中。下表列出了必须配置的字段：

参数名称 说明

AppsFlyer事件 事件的名称，即AppsFlyer从SDK侧接收到的事件名称或来自S2S事件的名称。
- 点击 添加项目 ，输入事件名称，然后点击 。

渠道映射事件 事件的转化操作ID需与合作渠道端保持一致。
- 在Google Ads 转化目标 页面，单击您创建的转化操作。

- 在转化操作 详情 页面，从URL参数 ctId= 中复制 转化操作ID 。

- 返回AppsFlyer并粘贴到对应位置。举例来说，假设 ctid=123456789 ，则将 123456789 粘贴到渠道映射事件字段中。

事件来源 仅该合作渠道： 仅回传归因到Meta Web的事件，也就是说完成该事件的用户来自Meta Web。

回传内容 发送值与收入： 在回传中发送所有事件参数，包括收入（若有）。

- 【可选】点击 添加条件 图标， 设置事件的回传条件 。

- 设置 应用内事件回传窗口 （如有需要）。详情请见 应用内事件回传窗口配置 。

- 点击 保存对接 。

- 【可选】您可以在保存对接后留在配置页面，将其他应用也对接到该渠道：
- 点击页面左上角渠道名称下方的应用名称，打开应用列表。

- 在下拉菜单中选择一个其他应用。

- 为选定应用完成对接设置。

### 归因链接选项卡

目前尚不支持从AppsFlyer后台生成归因链接。但您可以使用带有智能脚本的跨平台落地页，或为相关应用创建一条直接归因链接（见下表）。

可选方案 可用操作 人员

跨平台落地页
推荐 ：如果您要推广的应用在Epic和Steam等多个平台中上架，建议使用这种方式。

进一步了解跨平台落地页

了解跨平台落地页的设置方式

- 请确保所有相关URL参数（即智能脚本要抓取的参数）都已添加到广告链接中。

- 请确保智能脚本能从入链中抓取GCLID，并将其添加到生成的链接中。也就是说，传出链接GCLID（即智能脚本生成的链接）中的参数 gclid 必须映射到传入链接中 gclid 参数。 请注意 ：如需让AppsFlyer的原始数据报告呈现该数据，还必须将传入的 gclid 参数与 af_sub[1-5] 参数之一进行映射。
营销人员（可能需要开发人员协助）

直接归因链接
如果您要推广的应用仅在1个平台中上架，因而决定不使用跨平台落地页，可以创建直接归因链接，将其附在广告背后。

了解直接归因链接的创建方式

- 请务必在您的链接中添加下列参数：
- af_media_source

- af_campaign_id

- af_r
请注意 ：如果您使用Google Ads的 自动标记（auto-tagging） 功能，Google会自动将这个 Google点击标识符 参数添加到生成的链接中。如果您的Google Ads中禁用了自动标记功能，请在Google广告跟踪模板URL中添加 gclid={gclid} 。

- 配置af_camapign_id、af_adset_id和af_ad_id参数时，可以使用Google的 ValueTrack参数 (即动态URL参数）宏。所有其他参数都需要手动设置。

- 切勿添加af_dp参数。

适用于大多数平台（Epic、Steam、Xbox、PlayStation 等）的模板 如下：

https://engagements.appsflyer.com/v1.0/c2s/click/app/{platform}/{app_id}? af_media_source=googleads_int&af_campaign_id={campaignid}&af_campaign={campaignname}&af_adset={adgroupname}&af_adset_id={adgroupid}&af_ad_id={creative}&af_r={lpurl}

用于Steam的链接示例 ： https://engagements.appsflyer.com/v1.0/c2s/click/app/steam/123456?af_media_source=googleads_int&af_campaign_id={campaignid}&af_campaign=campaign1&af_adset=adgroup1&af_adset_id={adgroupid}&af_ad_id={creative}&af_r={lpurl}

仅适用于Google PC和Mac应用的模板 如下：

https://app.appsflyer.com/{app_id}?af_media_source=googleads_int&af_campaign_id={campaignid}&af_campaign={campaignname}&af_adset={adgroupname}&af_adset_id={adgroupid}&af_ad_id={creative}&af_r={lpurl}

用于Google PC的链接示例 ： https://app.appsflyer.com/123456?af_media_source=googleads_int&af_campaign_id={campaignid}&af_campaign=campaign1&af_adset=adgroup1&af_adset_id={adgroupid}&af_ad_id={creative}&af_r={lpurl}
营销人员

## 在Google Ads Web中设置PC和主机端应用的广告

设置全新的广告系列时，您可以选择将用户引导到：

- 跨平台落地页如果您要推广的应用在Epic和Steam等多个平台中上架，建议使用这种方式。

- 单一应用商店：使用直接归因链接，适用于仅在一个平台中上架的应用。

下文分别针对这两种方式，说明了相应的广告系列设置方法。

请注意 ：在设置广告系列的过程中，您可能需要根据自己的实际需求自行配置转化行为和目标。下文单开一节对这部分配置进行了说明，供您参考。

### 跳转到跨平台落地页的广告

如需让用户跳转到跨平台落地页，请按以下方式设置您的广告系列 ：

- 按 归因链接部分 的说明，创建一个跨平台落地页。

- 登入Google Ads并创建一个新的广告系列。

- 选择投放目标（campaign objective）。

- 除“App promotion”之外，所有目标均适用（“App promotion”仅适用于移动端应用的广告）。

- 建议您将投放目标设置为“Sales”，以便根据具体的应用内事件进行优化。

- 选择转化目标（conversion goals）。

- 默认情况下，您会看到账户层级的预定义投放目标（campaign goals）。您可以根据实际的投放需求保留或移除相关目标。

- 如果您没有在预定义列表中看到所需目标，请手动 添加相应的转化行为和转化目标 。

- 您还可以在Conversion页面中修改您的预定义目标。

- 选择所需的campaign type（广告系列类型）

- 除“Shopping”之外的所有广告系列类型均适用（“Shopping”类广告系列要求广告主配置商品目录，且仅限已验证的域名）。

- 输入跨平台落地页的网址，作为Google广告的跳转终点（Final URL）。

### 跳转到单一应用商店的广告

如需让用户跳转到某一个特定的应用商店，请按以下方式设置您的广告系列 ：

- 按 归因链接部分 的说明，创建一条直接归因链接。

- 登入Google Ads并创建一个新的广告系列。

- 选择投放目标（campaign objective）。

- 除“App promotion”之外，所有目标均适用（“App promotion”仅适用于移动端应用的广告）。

- 建议您将投放目标设置为“Sales”，以便根据具体的应用内事件进行优化。

- 选择转化目标（conversion goals）。

- 默认情况下，您会看到账户层级的预定义投放目标（campaign goals）。您可以根据实际的投放需求保留或移除相关目标。

- 如果您没有在预定义列表中看到所需目标，请手动 添加相应的转化行为和转化目标 。

- 您还可以在Conversion页面中修改您的预定义目标。

- 选择所需的campaign type（广告系列类型）

- 除“Shopping”之外的所有广告系列类型均适用（“Shopping”类广告系列要求广告主配置商品目录，且仅限已验证的域名）。

- 输入链接，作为Google广告的跳转终点（Final URL）。
- 请确保该网址是您需要引导用户进入的页面。您可以使用网站落地页或商店页面等的任何网址，例如： https://store.steampowered.com/app/{appid}/{myapp}/ 。

- 切勿使用AppsFlyer URL作为Final URL，否则可能会导致您的广告系列被拒审。请仅使用不带跳转的直接访问网址。

- 输入您的AppsFlyer直接归因链接，作为Google的跟踪模板（Tracking Template URL）。
- 为了保证并行追踪（Parallel tracking）能正常生效，请务必将跳转终点（Final URL）配置在Google的{lpurl} ValueTrack参数中（具体可参考Google的相关说明）。

- 切勿添加af_dp参数。

### 创建转化操作和目标

您可以从Google Ads预定义的转化行为（Conversion Actions）和目标（Goals）中选择，也可以创建自定义的行为和目标。

重要提示！ 如果需要AppsFlyer发送“First Open”（首次打开）事件回传，情在Google Ads中创建一个转化操作，并将其命名为“First Open”。

请按以下方式创建您自定义的Google Ads转化行为和目标 ：

- 进入您的Google Ads账户总览页面（Overview），从侧边栏中选择 Goals > Conversions > Summary 。

- 点击 + New conversion action 。

- 从转化类型（Conversion Types）列表中选择 Import ，然后选择 Manual import using API or uploads > Track conversions from clicks 。

#### 注意

如果无法 使用API或手动 上传：

- 选择 CRM、文件或其他数据源 > 基于点击追踪转化。

- 在 数据源 中选择 跳过此步骤，稍后设置数据源。

- 点击 继续 ，进入下一步。
界面会显示一张表格。

- 填写表格并点击 Create and continue ，完成转化追踪（Conversion Tracker）的设置。

- 在 Goal and action optimization 部分，选择一个类别（category），用于映射您设置的转化行为。
请注意 ：请注意：Google会按类别而非转化行为来优化投放。如果您只需针对某一个具体转化行为来优化投放，请仅将此转化行为映射到选定类别中。举例来说，假设您需要针对“install”（激活）行为来优化投放，请将“install”行为单独映射到其专用的类别中（如“Other”）。 然后您就可以在创建广告系列时选择这个类别作为投放目标。

- 从Google Ads复制转化操作ID，并将其输入到 映射到合作渠道事件 字段中。 转化操作ID 将作为合作渠道事件ID，用于在应用内事件回传设置中，将AppsFlyer事件名称映射到合作渠道事件ID。如需从Google Ads中复制转化操作ID，请执行以下操作：
- 在Google Ads 转化目标 页面，点击您创建的转化操作。

- 在转化操作 详细信息 页面，从URL参数 ctId= 中复制 转化操作ID 并粘贴。

- 点击 Create and continue（创建并继续） ，进入下一步。

- 点击 Finish ，完成设置。
