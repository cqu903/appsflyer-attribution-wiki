# PC和主机端应用的Meta Web对接设置

Source URL: https://support.appsflyer.com/hc/zh-cn/articles/18704273772689
Section: Meta ads
Updated: 2026-05-25T14:37:22Z

概要： 本文说明了PC和主机端应用通过AppsFlyer与Meta Web对接的方式，并介绍了Meta Web中的投放设置方式。

#### 重要提示！

此对接仅适用于PC或主机端的应用，包括Steam、Epic及PC端原生应用。如需了解适用于移动端应用的对接，请参阅 Meta ads对接设置 说明。

## 速览

本文针对使用AppsFlyer的PC和主机端应用，提供了Meta Web对接的设置和使用说明及其相关步骤，其中包含以下信息；

- 在AppsFlyer中启用Meta Web对接 。

- 在AppsFlyer中设置Meta Web对接 。

- 在Meta中为相关PC和主机端应用设置广告投放 。

### 前期准备

在启用Meta Web对接前，请确保您已在 Meta Business Manager 中开设了企业账户。

## 在AppsFlyer后台启用Meta Web

请按以下步骤启用该渠道：

- 在AppsFlyer面板的左侧菜单栏中点击 协作 > Partner Marketplace 。

- 搜索并选择 Meta Web 。

- 点击 设置对接 ，然后界面会跳转到 活跃对接 设置页面。

- 在 渠道对接 选项卡中，打开 启用该渠道 开关。
请注意： 在您与该渠道合作期间，请确保该开关总是处于打开（即 启用 ）状态。

- 参考下文说明，根据您的具体需求在相关选项卡中完成各项配置。

## 在AppsFlyer中设置Meta Web对接

这部分对接设置中包含Meta Web的回传配置。配置Meta Web（metaweb_int）对接后，广告主可以使用Meta ads提供的一个Conversions API工具从其 PC和主机端应用 直接向Meta服务器发送数据。这类数据包括用户在非移动端（包括网页和应用）完成的购买、注册等等各种转化行为。Meta ads会使用这些数据来优化广告的定向投放，衡量广告效果并创建用于再营销的自定义人群包。

Meta Web对接仅支持发送激活和应用内事件的回传。

### 渠道对接选项卡

渠道对接 选项卡中包括以下几个部分。

#### 启用渠道

在配置对接之前，必须先打开 启用该渠道 开关。

#### 通用设置

- Pixel ID： 输入相关的Pixe ID。您可以从Meta ads Manager后台获取该信息，具体方式如下：
- 在 Ads manager 侧边栏菜单中，向下滚动到 All tools > Events Manager 。
请注意： 请确保您在右上角的下拉菜单中选择的业务（Business）正确无误。

- 从 Events Manager 侧边栏菜单中选择 Data sources 。您会看到一个pixel列表。如果您选定的业务尚无pixel，可以在该窗口中创建一个pixel。

- 复制相关pixel名称下方的pixel ID，并将其粘贴到AppsFlyer后台对接页面的Pixel ID字段中。

- Access token（访问令牌）： 输入相关的token。您可以按照 这里的操作说明 来获取此信息。

- 限制Meta ads对用户个人信息的使用（CCPA）： 打开此开关可确保数据的使用方式符合相关的 数据保护法 规定。
请注意： 在CCPA隐私条例合规的情况下，来自美国加州Meta ads用户的数据会受到限制。

#### 启用浏览型归因

AppsFlyer支持通过启用 智能脚本 或 智能横幅 的落地页，基于概率性模型（PMOD）对Meta Web广告系列进行浏览型归因。因此，AppsFlyer能够将用户浏览广告带来的应用激活归因至Meta，即便用户未直接点击广告，也能在跨平台环境中识别。

当用户点击Meta Web广告并进入 智能脚本 或 智能横幅 落地页时，AppsFlyer可以将该活动与后续的应用安装进行匹配。若可用，AppsFlyer还会在给Meta的回传中包含 fbclid ，以提高归因精准度。

启用浏览型归因

- 在AppsFlyer中，从侧边栏打开 活跃对接 > Meta web ads 。

- 在 对接 选项卡中，下滑到 激活归因 部分。

- 打开 浏览型激活归因 开关。

- [可选] 开启 跨平台归因具有同等优先级 并设置回溯窗口。有关更多信息，请参见 为跨平台展示启用同等优先级 。

- 打开 归因链接 选项卡。

- 下滑到 浏览型归因 部分并设置 浏览归因回溯窗口 （最长24小时）。

- 点击 保存对接 以应用您的设置。

#### 默认回传

激活回传自动发送到Meta Web，不会回传给其他任何渠道。

#### 应用内事件回传

将回传中的应用内事件映射到Meta Web中的对应事件。您可以在 Meta Events Manager 中查看这些事件，具体方式如下：

- 打开 应用内事件回传 开关。

- 点击 添加事件 ，将SDK事件或S2S事件添加到列表中。下表列出了必须配置的字段：

参数名称 说明

AppsFlyer事件 事件的名称，即AppsFlyer从SDK侧接收到的事件名称或来自S2S事件的名称。
- 点击 添加项目 ，输入事件名称，然后点击 。

渠道映射事件 Meta Web为每个事件定义的专属名称或ID。
- 从下拉菜单中选择最合适的预定义渠道事件，或选择“自定义”。

事件来源 仅该合作渠道： 仅回传归因到Meta Web的事件，也就是说完成该事件的用户来自Meta Web。

回传内容 发送值与收入： 在回传中发送所有事件参数，包括收入（若有）。

- 【可选】点击 添加条件 图标， 设置事件的回传条件 。

- 设置 应用内事件回传窗口 （如有需要）。详情请见 应用内事件回传窗口配置 。

- 点击 保存对接 。

- 【可选】您可以在保存对接后留在配置页面，将其他应用也对接到该渠道：
- 点击页面左上角渠道名称下方的应用名称，打开应用列表。

- 在下拉菜单中选择一个其他应用。

- 为选定应用完成对接流程。

### 归因链接选项卡

目前尚不支持从AppsFlyer后台生成归因链接。但您可以使用带有智能脚本的跨平台落地页，或为相关应用创建一条直接归因链接（见下表）。

可选方案 可用操作 人员

跨平台落地页
推荐 ：如果您要推广的应用在Epic和Steam等多个平台中上架，建议使用这种方式。

进一步了解跨平台落地页

了解跨平台落地页的设置方式

- 请确保所有相关URL参数（即智能脚本要抓取的参数）都已添加到广告链接中。

- 请确保该脚本能从入链URL中抓取FBCLID，并将其添加到生成的链接中。也就是说，传出链接（即智能脚本生成的链接）中的 fbclid 参数必须映射到传入链接中 fbclid 参数。 请注意 ： 如需让AppsFlyer的原始数据报告呈现该数据，还必须将传入的参数 fbclid 映射到传出的参数之一 af_sub[1-5] 。

- 可选：使用Meta的动态URL宏，而非手动将参数值在代码中写死。 详情请见此处说明。
营销人员（可能需要开发人员协助）

直接归因链接
如果您要推广的应用仅在1个平台中上架，因而决定不使用跨平台落地页，可以创建直接归因链接，将其附在广告背后。

了解直接归因链接的创建方式

- 请务必在您的链接中添加下列参数：
- af_media_source

- af_campaign_id

- af_r

用于Meta Web广告链接的模板如下：

https://engagements.appsflyer.com/v1.0/c2s/click/app/{platform}/{app_id}?af_media_source=metaweb_int&af_campaign_id={{campaign.id}}&af_campaign={{campaign.name}}&af_adset={{adset.name}}&af_adset_id={{adset.id}}&af_ad={{ad.name}}&af_ad_id={{ad.id}}&af_r={encoded landing page and URL params}

用于Steam的Meta Web广告链接如下： https://engagements.appsflyer.com/v1.0/c2s/click/app/steam/123456?af_media_source=metaweb_int&af_campaign_id={{campaign.id}}&af_campaign={{campaign.name}}&af_adset={{adset.name}}&af_adset_id={{adset.id}}&af_ad={{ad.name}}&af_ad_id={{ad.id}}&af_r=https%3A%2F%2Fstore.steampowered.com%2Fapp%2F440%2FTest_app2%2F%3Futm_term%3Dhello

请注意 ：该模板中包含Meta支持的部分动态URL参数。您也可以手动使用实际值覆盖。 进一步了解Meta动态URL参数

### 成本选项卡

Meta Web的对接不支持成本数据。

### 广告收入选项卡

Meta Web的对接不支持广告收入数据。

### 授权选项卡

目前尚未向Meta Web开放权限。

## 在Meta Web中为相关PC和主机端应用设置广告投放

设置新的广告系列时请完成以下操作 ：

- 选择一个投放目标（campaign objective）。建议您使用以下任一选项：
- Sales ：该选项能让Meta针对某个具体事件进行优化（比如您的主要KPI事件）。

- Traffic ：该选项能让Meta针对广告点击进行优化。建议您仅在没有其他KPI或其他KPI事件的流量太低时使用该选项。

- Custom ：适用于需要将某个自定义事件设置为投放目标的场景。 请注意： 如需在Meta的面板中将自定义事件设置为投放目标，请先在AppsFlyer中配置该自定义事件。

- 将转化位置（conversion location）设置为 Website 。

- 为您的Facebook conversion pixel（转化追踪像素）命名时，请确保该名称与您在AppsFlyer Partner Marketplace中的metaweb_int对接下输入的pixel ID一致。

- 设置广告的跳转终点时，请务必使用以下任一网页的URL：
- 网站落地页（一个应用在多个平台中上架时建议使用该方式）

- 商店URL（相关应用仅在一个平台的一个应用商店中上架时建议使用该方式）
